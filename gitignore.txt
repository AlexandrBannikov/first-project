Игнорирование файлов в git.

Очень часто бывает что в папке репозитории есть файлы для которых не нужно хранить историю изменений, очень часто...

Чтобы Git игнорировал такие файлы и не пытался добавить их в репозиторий нужно создать файл .gitignore(англ. ignore - игнорировать), и записать в него названия игнорируемых файлов.

.gitignore -- это обычный текстовый файл.Его добавляют в корень репозитория и тоже коммитят.
В .gitignore указывают все файлы которые нужно игнорировать (по одному имени на строку).

Если строка начинается с #, то это комментарий, и гитигнор не будет его учитывать.
Допустим чтобы git начал игнорировать файлы .DS_Store для этого надо добавить в гитигнор строку с названием файла .DS_Store В этом случае git будет игнорировать все файлы с именем DS_Store, причем не только в корне репозитория но и во всех вложенных папках.
Символ * соответствует любой строке, включая пустую. *.jpeg # будут проигнорированны все файлы которые заканчиваются на .jpeg
# игнорировать все файлы tmp во всех подпапках папки docs -- docs/*/tmp например docs/current/tmp
Если задать правило которое состоит только из звездочки Git будет игнарировать все файлы.
# игнорировать все файлы -- *

Вопросительный знак ? -- соответствует одному любому символу
Если сохранить такую запись в гитигноре то файл fileQ.txt будет проигнорирован потому что один символ а вот файл fileQA.txt уже не будет проигнорирован потому что два символа.

Квадратные скобки ([]) -- тоже соответствуют одному символу, при этом символ не любой а только из списка file[0-2].txt -- будут проигнорированны файлы 0, 1, 2.txt
В скобках можно перечислять символы [adc], либо задавать диапазон [a-z]

Слеш (/) -- указывает на каталоги.
/todo.txt # будет игнорировать в корне репозитория, для сравнения spam.txt будет игнорироваться во всех папках.При этом файл subdir/todo.txt по прежнему отслеживается
Если шаблон заканчивается слешем то правило применится только к папке -- build/ #игнорировать папку build.

Парные звездочки (**) -- похожа на функцию одинарной но только отличие в том что она работает с вложенными папками. Двойная звездочка может соответствовать любому количеству таких папок(в том числе и нулю). Одинарная звездочка может соответствовать только одной.
docs/**/tmp # игнорируются файлы docs/current/tmp,а так же docs/old/saved/a/b/c/d/tmp
для двойной звездочки то же правило что и для одной, если задать правило ** то будут проигнорированны все файлы.

Восклицательный знак (!) -- инвертирует правила в гитигноре 
 # игнорировать все JPEG-файлы -- *.jpeg эти файлы будут проигнорированны
 # но только не мем с Doge  -- !doge.jpeg теперь этот файл будет отслеживаться
