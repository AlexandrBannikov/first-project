Создание и связывание ключа SSH 

SSH(англ. Secure Shell Protocol) один из наиболее распространенных сетевых протоколов. Он обеспечивает безопасный обмен данными в сети.С помощью этого протокола можно получать данные с удаленного компьютера или отправлять на него.Трафик шифруется, поэтому протокол безопасен.

SSH исполузует два ключа, приватный, который храниться только на вашем компьютере и публичный, доступен всем и используется для шифрования данных.

Проверить наличия у вас ssh ключ с помощью команды в основной директории ls -la .ssh/ # вывели список созданных ключей. Если есть файлы с похожими названиями id_ed25519.pub то ключи уже создавались.

Для генерации ssh пары можно использовать команду, ssh-keygen -t ed25519 -C "электронная почта, к которой привязан ваш аккаунт на GitHub" .

Проверить что ключи действительно сгенерировались можно командой, ls -a ~/.ssh .

Для привязки ключа и аккаунта на Гите нужно скопировать публичный ключ командой, pbcopy < ~/.ssh/id_rsa.pub .
Еще можно просмотреть публичный ключ командой, cat ~/.ssh/id_rsa.pub и скопировать его вручную и ввести на гите в окно для ключа.
В настройках аккаунта заходим в SSH, нажимаем new ssh, и вставляем ключ, нажимаем сохранить. В поле title нужно ввести название например Personal key.
Проверить правильность ключа можно с командой, ssh -T git@github.com .
